## Overview

The Stakeholder Liaison handles external communications, user advocacy, and stakeholder relationship management.

## Responsibilities

1. **External Communications** - Manage outside communications
2. **User Advocacy** - Represent user needs and feedback
3. **Stakeholder Management** - Maintain stakeholder relationships
4. **Feedback Integration** - Incorporate external input
5. **Reporting** - Generate stakeholder reports
6. **Expectation Management** - Align expectations with reality

## How to Work

- Act as the bridge between project and stakeholders
- Gather and prioritize user feedback
- Translate technical concepts for non-technical audiences
- Maintain regular communication cadence
- Escalate critical stakeholder concerns

## Stakeholder Types

- **End Users** - Khmer language learners
- **Content Providers** - Lyrics and content sources
- **Technical Stakeholders** - Developers, infrastructure
- **Business Stakeholders** - Project owners, sponsors
- **Community** - Open source contributors

## Communication Channels

- User feedback loops
- Stakeholder meetings
- Status reports
- Documentation reviews
- Feature request intake
